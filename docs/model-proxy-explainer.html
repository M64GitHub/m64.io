Model Proxy

A utility meter and breaker panel for your agents' model calls.

Every LLM call your fleet makes — forwarded untouched, metered exactly, governable — through one environment variable.

In one linePoint an agent's base URL at the proxy and every model call is forwarded byte-for-byte while an exact metering record — provider, model, tokens, latency, identity — lands in JetStream; budgets, model policy, and per-agent keys enforce on those same records without ever slowing a call.

01 — The problem

Your agents spend real money through a pipe you can't see.

Fleets of heterogeneous agents — Claude Code, Codex, custom harnesses — call model providers over opaque HTTPS.

Token spend is invisible until the invoice, and the invoice doesn't say which agent. Attribution is guesswork. And when an agent loops at 3 a.m., nothing stands between it and the bill.

The Agent Fabric observes agents through three lenses over one NATS substrate — and model calls were the blind spot, because they're the one thing agents do that doesn't speak NATS:

the wire · agent-audit· cognition & cost · this proxy· the work · durable-execution

The proxy is the fabric's on-ramp for model traffic — and, since Phase 2, its breaker panel.

02 — Kill the wrong box

Deliberately the dumbest possible proxy — with the best possible bookkeeping.

Two mental models will mislead you here. Both die now.

This is not an LLM gateway, and the metering middleman cannot hurt your traffic. The proxy sits in the model-call critical path — the product's only fail-closed component — so its reliability comes from scope discipline, not features.

The cleverness lives in the bookkeeping, never in the pipe. That's the design.

03 — The contract

Onboarding is an env var. Leaving is unsetting it.

Identity rides the URL path, so any SDK that honors a base URL is already integrated — zero code changes.

In practice:

onboarding.envthe whole integration
# any OpenAI-compatible SDK
OPENAI_BASE_URL=http://proxy:8080/m64/codex/dev/openai/v1
# the Anthropic SDK appends /v1/messages itself
ANTHROPIC_BASE_URL=http://proxy:8080/m64/claude-code/dev/anthropic
The bypass guaranteeUnset the variable and traffic goes direct to the provider. The proxy holds no keys (in transparent mode) and no state an agent depends on — never a lock-in point.
04 — The data path

Two lanes: one sacred, one fail-open.

Forwarding streams byte-for-byte with immediate flush. A passive tee feeds the meter.

The metering pipeline is the two-stage fail-open ring transplanted from synadia-agent-audit: a non-blocking in-memory sink (drop-and-count on overflow, its channel never closes), then a JetStream publisher on its own NATS connection that retries forever and survives a down server at startup. Records publish to audit.llm.<type>.<agent>.<owner>.<session>.

05 — Exact tokens

The provider's numbers — or an honest none.

No tokenizer estimates, ever. "Exact from day one" means the provider's own usage object, or usage_source: "none", counted.

  • Anthropic streams report usage natively (message_start / message_delta) — that path never mutates anything.
  • Non-streaming responses (both shapes) carry usage in the body.
  • OpenAI streams with include_usage append one final usage chunk.
  • OpenAI streams without it report nothing at all — verified live against Ollama. And that's the dominant call shape in real fleets.

Inject + strip — the one sanctioned mutation

For streaming requests that didn't opt in, the proxy asks the provider for the number — and erases the question:

1
Inject stream_options.include_usage: true — a value-preserving rewrite: only the top level is re-encoded, so every other field keeps its exact bytes (int64 seeds and long floats included; pinned by test).
2
The provider appends one extra usage chunk the client's SDK does not expect.
3
Strip exactly that chunk (choices == [], usage != null) from the client-visible stream. At most one event is ever buffered.
4
The record says "mutated": true. If a stream stops looking like well-formed SSE, the filter degrades to verbatim pass-through — worst case the client sees one extra chunk. Forwarding first, always.
06 — What lands in the stream

One document per call. No payloads. No secrets — by construction.

A real record, captured from the live loop (an OpenAI-SDK agent → proxy → Ollama):

Records are typed — llm.call (generation endpoints, token-parsed) and llm.http (everything else: status/latency/bytes) — and land in the LLM_USAGE stream, a 30-day discard-old ring. usage_raw keeps the provider's object verbatim; on OpenRouter that includes their USD cost — provider-priced billing data for free.

07 — Phase 2 · trust & policy

Your records become your enforcement.

Budgets, model allow/deny, and per-agent keys — all opt-in, all built on the records the proxy already emits.

The core invariant survived Phase 2 intact: the request path does zero policy I/O. Enforcement answers from in-memory mirrors fed by KV watches; a dead policy store can only make enforcement stale (last-known state keeps answering, the D1 "stale-allow" posture; --policy-strict flips it), never slow a model call.

Denials SDKs respect

Budgets are eventually-consistent guards with an observable staleness bound (spend_age_s in healthz). The hard boundary is the key, not the budget.

08 — Per-agent keys · the vault

The agent never holds the real key. The store never holds a usable one.

1
The agent presents sk-syn-v1-… as its ordinary API key. The proxy stores only its SHA-256 — a KV compromise leaks nothing usable.
2
Identity binding: the key's {owner · agent} must match the URL path — 403 key_identity_mismatch stops key borrowing across agents. Optional per-key provider allowlists.
3
The proxy unseals the provider key — AES-256-GCM under a vault key that never enters NATS — and swaps auth per provider shape. The virtual key never goes upstream; the real key never comes down; records carry vk_id, never material.
4
Revocation is a KV delete — applied on the next KV-watch tick, typically milliseconds (a frozen mirror during a store outage keeps last-known keys, by design). Wrong or missing vault key fails closed (503) for virtual traffic only; transparent traffic is never affected.
09 — The proof story

Adversarially tested — then proven live.

Every feature ships with its failure modes exercised: crash, disconnect, concurrency, failure injection — never just the happy path.

conservation

Every emit is written or counted

Exactly once, under 64-way race — the agent-audit invariants ported as executable philosophy. The sink's channel never closes; a late emit never panics.

outage mid-load

JetStream killed under traffic

45 calls through a mid-run broker kill and restart: forwarding latency flat, 44 records durable, 1 counted ack-error. The pipeline reconnects and re-syncs by itself.

the harness bites

It found a real bug

A buffered-SSE provider (legal, unusual) exposed a stale Content-Length on stripped streams — clients saw truncation. Fixed, regression-pinned.

live exactness

Record == what the client saw

OpenRouter usage matched the record field-by-field; the Ollama loop runs fully offline; a virtual key completed against the real API with the sealed key swapped in.

The whole gate — go build ./... && go vet ./... && go test -race ./... — runs offline: integration tests spawn their own nats-server -js and skip when tools are absent. 9 packages, green under the race detector.
